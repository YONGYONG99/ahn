# 01. 인터넷과 HTTP

## 웹 서비스의 기본 흐름

우리가 브라우저에서 `www.google.com` 을 입력하면 내부적으로 이런 일이 일어납니다.

```
[브라우저]  →  "구글 홈페이지 줘!"  →  [구글 서버]
[브라우저]  ←  "여기 있어 (HTML)"  ←  [구글 서버]
```

이 대화의 규칙(프로토콜)이 바로 **HTTP** 입니다.

---

## HTTP 란?

**HyperText Transfer Protocol**의 약자로,  
클라이언트(브라우저)와 서버가 데이터를 주고받는 **약속된 규칙**입니다.

- **클라이언트**: 요청을 보내는 쪽 (브라우저, 앱, Next.js 등)
- **서버**: 요청을 받아 처리하고 응답하는 쪽 (Django 등)

---

## 요청(Request)과 응답(Response)

### 요청(Request) — 클라이언트가 서버에게 보내는 것

```
GET /users/1 HTTP/1.1
Host: api.example.com
Authorization: Bearer abc123
```

요청은 크게 3가지로 구성됩니다:

| 구성요소 | 설명 | 예시 |
|---|---|---|
| **메서드** | 뭘 하고 싶은지 | `GET`, `POST`, `DELETE` |
| **URL** | 어디에 요청하는지 | `/users/1` |
| **헤더** | 부가 정보 | 인증 토큰, 데이터 형식 등 |
| **바디** | 전달할 데이터 (선택) | 회원가입 시 이름, 이메일 등 |

---

### 응답(Response) — 서버가 클라이언트에게 돌려주는 것

```
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 1,
  "name": "홍길동",
  "email": "hong@example.com"
}
```

응답도 크게 3가지로 구성됩니다:

| 구성요소 | 설명 | 예시 |
|---|---|---|
| **상태코드** | 요청이 잘 됐는지 | `200 OK`, `404 Not Found` |
| **헤더** | 응답 부가 정보 | 데이터 형식, 인코딩 등 |
| **바디** | 실제 데이터 | JSON, HTML 등 |

---

## HTTPS 는 뭔가요?

HTTP에 **S(Secure)** 가 붙은 것으로, 데이터를 암호화해서 주고받습니다.  
요즘 웹 서비스는 거의 HTTPS를 씁니다. 개념적으로는 HTTP와 동일합니다.

---

## 정리

```
클라이언트(Next.js)
    │
    │  ① HTTP 요청 (메서드 + URL + 헤더 + 바디)
    ▼
서버(Django)
    │
    │  ② HTTP 응답 (상태코드 + 헤더 + 바디)
    ▼
클라이언트(Next.js)
    │
    ▼
화면에 데이터 표시
```
